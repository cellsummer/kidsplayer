# Thumbnails Folder

Place your thumbnail images here.

## File Naming

Name your thumbnail files to match the song IDs in the main `index.html` file:

- `song1.jpg`
- `song2.jpg`
- `song3.jpg`
- `song4.jpg`
- etc.

## Supported Formats

- **JPG/JPEG** (recommended)
- PNG
- GIF
- WebP
- JIFF/JFIF

## Recommended Specifications

- **Size**: 500x500 pixels or larger (square format works best)
- **Aspect Ratio**: 1:1 (square) or 4:3
- **File Size**: Keep under 500KB per image for faster loading
- **Quality**: Use bright, colorful, high-contrast images that are easy to see

## Example

If you have a song configured in `index.html` as:

```javascript
{ id: 'happy-birthday', title: 'Happy Birthday' }
```

Then your thumbnail file should be named:
```
happy-birthday.jpg
```

**Note:** You can use `.jpg`, `.png`, `.svg`, `.gif`, `.jiff`, or `.jfif` extensions.

## Tips for Kid-Friendly Thumbnails

- Use bright, vibrant colors
- Choose simple, recognizable images
- Consider using cartoon characters or icons
- Avoid dark or complex images
- Make sure images are child-appropriate
- Use high contrast for better visibility
- Consider adding text or song titles to the image itself

## Missing Thumbnails

If a thumbnail is missing, the app will automatically show a colorful placeholder with a music note emoji 🎵, so the app will still work even without images.

## Creating Thumbnails

You can create thumbnails using:
- Online tools (Canva, Photopea, etc.)
- Image editing software (GIMP, Photoshop, etc.)
- Screenshots from music videos
- Album artwork
- Custom illustrations
